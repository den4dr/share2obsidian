## REQ-001: 全コンテンツタイプで編集画面表示 🔵

**信頼性**: 🔵 *ユーザヒアリング（常に表示）より*

### Given
- Share2Obsidian が `ACTION_SEND` インテントを受信した

### When
- テキスト・URL（抽出完了後）・HTML・ファイルのいずれかのコンテンツ処理が完了した

### Then
- Obsidian を起動せずに編集画面が表示される

### テストケース

#### 正常系

- [ ] **TC-001-01**: テキスト共有時に編集画面が表示される 🔵
  - **入力**: `ACTION_SEND`, `text/plain`, `EXTRA_TEXT = "サンプルテキスト"`
  - **期待結果**: 編集画面が表示され、本文フィールドに "サンプルテキスト" が入力済み
  - **信頼性**: 🔵 *ユーザヒアリングより*

- [ ] **TC-001-02**: HTML 共有時に編集画面が表示される 🔵
  - **入力**: `ACTION_SEND`, `text/html`, `EXTRA_HTML_TEXT = "<h1>タイトル</h1>"`
  - **期待結果**: 編集画面が表示され、本文フィールドに Markdown 変換済み内容が入力済み
  - **信頼性**: 🔵 *ユーザヒアリングより*

- [ ] **TC-001-03**: ファイル共有時に編集画面が表示される 🔵
  - **入力**: `ACTION_SEND`, `image/*`, `EXTRA_STREAM = <Uri>`
  - **期待結果**: クリップボードコピー後に編集画面が表示される
  - **信頼性**: 🔵 *ユーザヒアリングより*

## REQ-003 + REQ-004: 編集フィールドとボタン表示 🔵

**信頼性**: 🔵 *ユーザヒアリング（編集項目・ボタン構成）より*

### Given
- 編集画面が表示されている

### When
- 画面を視認する

### Then
- タイトルフィールドが表示されている
- 本文フィールド（複数行）が表示されている
- タグフィールドが表示されている
- フォルダフィールドが表示されている
- 「送信」ボタンが表示されている
- 「キャンセル」ボタンが表示されている

### テストケース

#### 正常系

- [ ] **TC-003-01**: タイトル初期値が ProcessedContent.title で設定される 🔵
  - **入力**: `EXTRA_SUBJECT = "テストページ"` を含むインテント
  - **期待結果**: タイトルフィールドに "テストページ" が初期表示
  - **信頼性**: 🔵 *既存実装より*

- [ ] **TC-003-02**: タイトルが null の場合、タイトルフィールドは空 🟡
  - **入力**: `EXTRA_SUBJECT` なしのインテント
  - **期待結果**: タイトルフィールドが空文字で初期表示
  - **信頼性**: 🟡 *ProcessedContent.title nullable 対応から推測*

- [ ] **TC-003-03**: タグフィールドの初期値が AppConfig.OBSIDIAN_TAGS から生成される 🔵
  - **入力**: `AppConfig.OBSIDIAN_TAGS = listOf("shared")`
  - **期待結果**: タグフィールドに "shared" が初期表示
  - **信頼性**: 🔵 *既存 AppConfig より*

- [ ] **TC-003-04**: フォルダフィールドの初期値が AppConfig.OBSIDIAN_FOLDER 🔵
  - **入力**: `AppConfig.OBSIDIAN_FOLDER = "70_clippings"`
  - **期待結果**: フォルダフィールドに "70_clippings" が初期表示
  - **信頼性**: 🔵 *既存 AppConfig より*

## REQ-101: 送信時の Obsidian 起動 🔵

**信頼性**: 🔵 *ユーザヒアリング・既存実装より*

### Given
- 編集画面が表示されている
- ユーザーが各フィールドを編集済み（または初期値のまま）

### When
- 「送信」ボタンをタップする

### Then
- 編集後の値で Frontmatter が生成される
- 編集後の値で Obsidian URI が構築される
- Obsidian アプリが起動する
- `finish()` が呼ばれる

### テストケース

#### 正常系

- [ ] **TC-101-01**: 編集後の内容で Obsidian URI が構築される 🔵
  - **入力**: タイトル "編集後タイトル"、本文 "編集後本文"、タグ "shared, web"、フォルダ "notes"
  - **期待結果**: Obsidian URI に `title=編集後タイトル`、`content=---\ntitle: "編集後タイトル"\ntags: [shared, web]\n---\n\n編集後本文`、`folder=notes` が含まれる
  - **信頼性**: 🔵 *既存 FrontmatterBuilder・ObsidianUriBuilder より*

- [ ] **TC-101-02**: タグのカンマ区切り入力が List に変換される 🔵
  - **入力**: タグフィールド = "shared, web, clipping"
  - **期待結果**: Frontmatter に `tags: [shared, web, clipping]` が出力される
  - **信頼性**: 🔵 *ユーザヒアリング（カンマ区切り）より*

- [ ] **TC-101-03**: タグのカンマ区切りでスペースがトリムされる 🔵
  - **入力**: タグフィールド = "shared ,  web , clipping "
  - **期待結果**: Frontmatter に `tags: [shared, web, clipping]` が出力される（スペース除去）
  - **信頼性**: 🔵 *REQ-103 より*

#### 異常系

- [ ] **TC-101-E01**: Obsidian 未インストール時にトーストが表示される 🔵
  - **入力**: 送信ボタンタップ、Obsidian 未インストール
  - **期待結果**: `ActivityNotFoundException` がキャッチされ、トーストが表示され `finish()` が呼ばれる
  - **信頼性**: 🔵 *既存 REQ-401 実装より*

## REQ-301: URL処理後に編集画面へ遷移 🔵

**信頼性**: 🔵 *ユーザヒアリング（ローディング後に編集画面）より*

### Given
- `text/plain` かつ URL パターンのインテントを受信した

### When
- WebView による本文抽出が完了した（またはタイムアウト）

### Then
- 編集画面に遷移する
- 本文フィールドに抽出テキスト（またはURL）が設定されている

### テストケース

- [ ] **TC-301-01**: URL 抽出完了後に編集画面が表示される 🔵
  - **入力**: `EXTRA_TEXT = "https://example.com"`, WebView が本文を返す
  - **期待結果**: ローディング画面が消え、編集画面が表示される
  - **信頼性**: 🔵 *ユーザヒアリングより*

- [ ] **TC-301-02**: URL 抽出タイムアウト時に URL テキストで編集画面が表示される 🟡
  - **入力**: `EXTRA_TEXT = "https://example.com"`, WebView タイムアウト
  - **期待結果**: 本文フィールドに "https://example.com" が入力された編集画面が表示される
  - **信頼性**: 🟡 *REQ-302 フォールバック動作から推測*
