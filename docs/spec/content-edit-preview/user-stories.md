## エピック1: 編集画面の表示・操作

### ストーリー 1.1: テキスト共有時の編集 🔵

**信頼性**: 🔵 *ユーザヒアリング（常に表示）・既存実装より*

**私は** Share2Obsidian ユーザー **として**
**テキストを他アプリから共有したとき、送信前に内容を確認・修正したい**
**そうすることで** 意図した内容のみを Obsidian に保存できる

**関連要件**: REQ-001, REQ-002, REQ-003, REQ-004

**詳細シナリオ**:
1. 他アプリ（ブラウザ、メモアプリ等）でテキストを選択し「共有」をタップ
2. Share2Obsidian が選択される
3. テキスト処理完了後、編集画面が表示される
4. タイトル・本文・タグ・フォルダが初期値で入力済みになっている
5. ユーザーが必要に応じて各フィールドを編集
6. 「送信」ボタンをタップ → Obsidian が起動し編集後の内容でノートが作成される

**前提条件**:
- Share2Obsidian がインストール済み
- Obsidian がインストール済み

**制約事項**:
- Compose でシングルアクティビティ内に実装

**優先度**: Must Have

---

### ストーリー 1.2: URL 共有時の編集 🔵

**信頼性**: 🔵 *ユーザヒアリング（ローディング後に編集画面）より*

**私は** Share2Obsidian ユーザー **として**
**URL を共有したとき、WebView で取得されたページ本文を確認・編集してから Obsidian に保存したい**
**そうすることで** 不要な部分を削除したり、タイトルを整えたりしてから保存できる

**関連要件**: REQ-001, REQ-301, REQ-302

**詳細シナリオ**:
1. ブラウザで URL を共有する
2. Share2Obsidian の URL ローディング画面（`LoadingScreen`）が表示される
3. WebView でページ本文を抽出完了
4. 編集画面に遷移。本文フィールドには抽出されたテキストが入力済み
5. ユーザーが本文を整形・不要部分を削除
6. 「送信」ボタンをタップ → Obsidian が起動

**前提条件**:
- インターネット接続がある
- URL が有効なページを指している

**優先度**: Must Have

---

### ストーリー 1.3: HTML 共有時の編集 🔵

**信頼性**: 🔵 *既存 HtmlContentProcessor 実装・ユーザヒアリングより*

**私は** Share2Obsidian ユーザー **として**
**Web ページを「共有」で送ったとき、HTML から変換された Markdown 本文を確認・修正してから Obsidian に保存したい**
**そうすることで** Markdown 変換の品質を確認してから保存できる

**関連要件**: REQ-001, REQ-003

**詳細シナリオ**:
1. ブラウザで「共有」を選択（`text/html` インテント）
2. HTML → Markdown 変換完了後、編集画面が表示される
3. 本文フィールドに変換された Markdown が表示されている
4. 必要に応じて編集して送信

**優先度**: Must Have

---

### ストーリー 1.4: 送信キャンセル 🔵

**信頼性**: 🔵 *ユーザヒアリング（キャンセルで finish）より*

**私は** Share2Obsidian ユーザー **として**
**誤って共有してしまったとき、Obsidian に送信せずにアプリを閉じたい**
**そうすることで** 不要なノートが作成されることを防げる

**関連要件**: REQ-201

**詳細シナリオ**:
1. 誤って Share2Obsidian を選択してしまった
2. 編集画面で「キャンセル」ボタンをタップ
3. アプリが終了（Obsidian は起動しない）

**前提条件**: なし

**優先度**: Must Have

## エピック2: フィールド編集

### ストーリー 2.1: タイトルの編集 🔵

**信頼性**: 🔵 *ユーザヒアリング（タイトル編集）より*

**私は** Share2Obsidian ユーザー **として**
**Obsidian に保存するノートのタイトルを共有前に変更したい**
**そうすることで** より分かりやすい名前でノートを整理できる

**関連要件**: REQ-003, REQ-101

**詳細シナリオ**:
1. 編集画面でタイトルフィールドをタップ
2. キーボードが表示される
3. 既存のタイトルを変更
4. 送信ボタンをタップ → 変更後のタイトルが Frontmatter と URI に使用される

**優先度**: Must Have

---

### ストーリー 2.2: タグの編集 🔵

**信頼性**: 🔵 *ユーザヒアリング（タグ編集・カンマ区切り）より*

**私は** Share2Obsidian ユーザー **として**
**保存するノートに追加のタグを付けたい**
**そうすることで** Obsidian での検索・分類が容易になる

**関連要件**: REQ-003, REQ-103

**詳細シナリオ**:
1. 編集画面でタグフィールドを確認（初期値: `shared`）
2. カンマ区切りでタグを追加（例: `shared, web, tech`）
3. 「送信」をタップ
4. Frontmatter に `tags: [shared, web, tech]` として出力される

**優先度**: Must Have

---

### ストーリー 2.3: 保存先フォルダの変更 🔵

**信頼性**: 🔵 *ユーザヒアリング（フォルダ編集）より*

**私は** Share2Obsidian ユーザー **として**
**この共有コンテンツだけを別のフォルダに保存したい**
**そうすることで** デフォルトと異なるフォルダへの一時的な振り分けができる

**関連要件**: REQ-003, REQ-405

**詳細シナリオ**:
1. 編集画面でフォルダフィールドを確認（初期値: `AppConfig.OBSIDIAN_FOLDER`）
2. フォルダ名を変更（例: `70_clippings/web`）
3. 「送信」をタップ → 変更後のフォルダへ保存される

**優先度**: Should Have
