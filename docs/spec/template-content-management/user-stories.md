## エピック1: テンプレートで本文構造を定義する

### ストーリー 1.1: 本文テンプレートの作成 🔵

**信頼性**: 🔵 *PRD「フィールドは本文含めてすべてテンプレート内で指定可能」・ユーザヒアリングより*

**私は** Share2Obsidian ユーザー **として**
**テンプレート編集画面で本文の初期構造を定義したい**
**そうすることで** 共有するたびに決まったMarkdown構造（セクション見出しや補足メモ欄など）を自動で付与できる

**関連要件**: REQ-002, REQ-052, REQ-053

**詳細シナリオ**:
1. TemplateEditScreen を開く
2. 本文テンプレート入力エリアに以下のようなテキストを入力する:
   ```
   ## 共有内容
   {{content}}

   ## 感想・メモ

   ```
3. テンプレートを保存する
4. 次回の共有時にこのテンプレートが適用され、EditScreen の本文欄に上記構造が表示される

**前提条件**:
- テンプレートが作成済みであること

**制約事項**:
- `{{content}}` プレースホルダーは任意（含まなくてもよい）

**優先度**: Must Have

---

### ストーリー 1.2: `{{content}}` プレースホルダーによる共有コンテンツ埋め込み 🔵

**信頼性**: 🔵 *ユーザヒアリング「{{content}}プレースホルダーで共有内容を埋め込む」より*

**私は** Share2Obsidian ユーザー **として**
**テンプレートの `{{content}}` の位置に共有したコンテンツが自動で入ってほしい**
**そうすることで** 共有内容の前後に定型文や見出しを付けた構造的なノートを作れる

**関連要件**: REQ-011, REQ-012

**詳細シナリオ**:
1. body = `"## 記事\n{{content}}\n\n## コメント\n"` のテンプレートが適用される
2. URLを共有した際に WebView で本文が抽出される
3. EditScreen の本文欄には `"## 記事\n{URL本文}\n\n## コメント\n"` と表示される

**前提条件**:
- デフォルトテンプレートに `body` が設定されていること
- URLの共有である場合、WebView 本文抽出が完了していること

**優先度**: Must Have

---

### ストーリー 1.3: 本文テンプレート未設定時の後方互換動作 🔵

**信頼性**: 🔵 *ユーザヒアリング「共有コンテンツをそのまま使用」より*

**私は** Share2Obsidian ユーザー **として**
**テンプレートに本文が未設定の場合でも従来通り共有コンテンツが本文欄に表示されてほしい**
**そうすることで** テンプレートを使い始めても既存の動作が壊れない

**関連要件**: REQ-013, REQ-014

**詳細シナリオ**:
1. body = `""` のテンプレートが適用される（または無テンプレート）
2. URLを共有した際に WebView で本文が抽出される
3. EditScreen の本文欄には共有コンテンツがそのまま表示される

**優先度**: Must Have

## エピック2: 保存先（vault/folder）の編集フロー

### ストーリー 2.1: EditScreen で保存ごとに vault/folder を設定 🔵

**信頼性**: 🔵 *PRD「保存先の内容は保存時に都度設定する前提」・ユーザヒアリングより*

**私は** Share2Obsidian ユーザー **として**
**共有のたびに Obsidian の vault や保存フォルダを確認・変更したい**
**そうすることで** コンテンツの種類に応じて適切な保存先に振り分けられる**

**関連要件**: REQ-022, REQ-023, REQ-024, REQ-041, REQ-043

**詳細シナリオ**:
1. URL を別アプリから Share2Obsidian に共有する
2. EditScreen が表示される。最上部に vault 欄、その次に folder 欄が表示され、DataStore のデフォルト値で初期化されている
3. 今回は別のフォルダに保存したいので folder 欄を変更する
4. 送信すると変更後の vault/folder が使われた Obsidian URI が開く
5. DataStore のデフォルト値は変わっていない

**前提条件**:
- DataStore に vault/folder のデフォルト値が設定されていること（空欄でも可）

**優先度**: Must Have

---

### ストーリー 2.2: テンプレートに vault/folder が含まれず内容のみを定義 🔵

**信頼性**: 🔵 *PRD「テンプレートに紐付くのはあくまでファイルの内容のみ」より*

**私は** Share2Obsidian ユーザー **として**
**テンプレートを「どこに保存するか」ではなく「何を書くか」の定義として使いたい**
**そうすることで** 同じコンテンツ構造を使いながら保存先を柔軟に変えられる**

**関連要件**: REQ-001, REQ-031, REQ-051

**詳細シナリオ**:
1. TemplateEditScreen を開く
2. 画面には vault / folder の入力欄がない
3. テンプレート名、frontmatter フィールド、本文テンプレートのみを設定して保存する
4. このテンプレートを適用しても EditScreen の vault/folder は DataStore の値から初期化される

**優先度**: Must Have

## エピック3: EditScreen の表示順変更

### ストーリー 3.1: 情報の重要度に沿った表示順で確認・編集 🔵

**信頼性**: 🔵 *PRD「保存画面では、上から順に、vault,フォルダ, frontmatter, 本文の順」より*

**私は** Share2Obsidian ユーザー **として**
**EditScreen で保存先 → タイトル → メタ情報 → 本文の順に情報を確認したい**
**そうすることで** 最も重要な「どこに何のファイルとして保存するか」を最初に確認できる**

**関連要件**: REQ-041, REQ-042

**詳細シナリオ**:
1. 共有後 EditScreen が表示される
2. 最上部: vault 欄（DataStore 値で初期化）
3. 次: folder 欄（DataStore 値で初期化）
4. 次: title 欄（ファイル名として明示、frontmatter とは別セクション）
5. 次: frontmatter フィールド（カスタムフィールドのリスト）
6. 最下部: 本文（テンプレート適用済み）

**優先度**: Must Have
