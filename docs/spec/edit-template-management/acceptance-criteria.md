## REQ-001 / REQ-011: TemplateListScreen ナビゲーション 🔵

**信頼性**: 🔵 *ユーザヒアリング・REQ-001, REQ-011 より*

### Given
- SettingsScreen が表示されている

### When
- ユーザーが「テンプレート管理」メニューをタップする

### Then
- TemplateListScreen に遷移する
- テンプレートの一覧が表示される（0件の場合は空状態表示）

### テストケース

#### 正常系

- [ ] **TC-001-01**: SettingsScreen からの TemplateListScreen 遷移 🔵
  - **入力**: SettingsScreen の「テンプレート管理」タップ
  - **期待結果**: TemplateListScreen に遷移、戻るボタンで SettingsScreen に戻る

- [ ] **TC-001-02**: テンプレート 0 件の空状態表示 🟡
  - **入力**: テンプレートが存在しない状態で TemplateListScreen を開く
  - **期待結果**: 空状態のメッセージまたはイラストが表示される

- [ ] **TC-001-03**: テンプレート複数件の一覧表示 🔵
  - **入力**: 複数テンプレートが存在する状態で TemplateListScreen を開く
  - **期待結果**: すべてのテンプレートが一覧に表示される

## REQ-012〜REQ-015: テンプレート作成・属性 🔵

**信頼性**: 🔵 *PRD・ユーザヒアリングより*

### Given
- TemplateListScreen が表示されている

### When
- 「+」ボタンをタップし TemplateEditScreen（新規モード）でテンプレートを入力・保存する

### Then
- テンプレートが Room DB に保存される
- TemplateListScreen に戻り、新規テンプレートが一覧に表示される

### テストケース

#### 正常系

- [ ] **TC-012-01**: 必須属性を入力してテンプレートを保存 🔵
  - **入力**: 名前="Web記事", vault="testVault", folder="70_clippings"
  - **期待結果**: Room DB に保存され一覧に表示される

- [ ] **TC-012-02**: vault・folder 未入力でも保存可能 🟡
  - **入力**: 名前="シンプル", vault="", folder=""
  - **期待結果**: 保存成功（空文字はデフォルト値として扱う）

#### 異常系

- [ ] **TC-012-E01**: テンプレート名が空の場合のバリデーション 🔵
  - **入力**: 名前=""（空）
  - **期待結果**: 保存できず、名前必須のエラー表示

- [ ] **TC-012-E02**: テンプレート名が既存と重複する場合のバリデーション 🔵
  - **前提**: 名前="Web記事" のテンプレートが既に存在する
  - **入力**: 新規テンプレートに名前="Web記事" を入力して保存しようとする
  - **期待結果**: 保存できず「同じ名前のテンプレートが既に存在します」のエラー表示
  - **信頼性**: 🔵 *ユーザヒアリング「重複不可（バリデーションで弾く）」確認済み*

## REQ-031〜REQ-033: フィールド追加・削除 🔵

**信頼性**: 🔵 *PRD「front matterの項目を追加・削除できる機能」より*

### テストケース

#### 正常系

- [ ] **TC-031-01**: フィールドの追加 🔵

## REQ-041〜REQ-045: 値ソース種別 🔵

**信頼性**: 🔵 *ユーザヒアリング 4種類の値ソースより*

### テストケース

#### FIXED（固定値）

- [ ] **TC-041-01**: FIXED 文字列フィールドが EditScreen に初期値として表示される 🔵
  - **前提**: FIXED, STRING, 初期値="draft" のフィールドをデフォルトテンプレートに設定
  - **入力**: URL を共有
  - **期待結果**: EditScreen のカスタムフィールドに `status: draft` が表示される

- [ ] **TC-041-02**: FIXED リスト型フィールドが Frontmatter に正しく出力される 🔵
  - **前提**: FIXED, LIST, 初期値="web, reading" のフィールド
  - **期待結果**: `tags: [web, reading]` 形式で Frontmatter に出力される

#### HTML_META（HTML メタデータマッピング）

- [ ] **TC-042-01**: og:title マッピングが URL 共有時に反映される 🔵
  - **前提**: HTML_META, OG_TITLE のフィールドをデフォルトテンプレートに設定
  - **入力**: og:title="テスト記事" を持つページの URL を共有
  - **期待結果**: EditScreen のカスタムフィールドに `フィールドキー: テスト記事` が表示される

- [ ] **TC-042-02**: og:description マッピングが URL 共有時に反映される 🔵
  - **入力**: og:description を持つページの URL を共有
  - **期待結果**: カスタムフィールドに説明文が自動入力される

- [ ] **TC-042-03**: URL マッピングが反映される 🔵
  - **入力**: URL を共有
  - **期待結果**: カスタムフィールドに URL 文字列が自動入力される

- [ ] **TC-042-04**: 公開日マッピング（article:published_time）が反映される 🔵
  - **入力**: article:published_time を持つページの URL を共有
  - **期待結果**: カスタムフィールドに公開日が自動入力される

- [ ] **TC-042-05**: 著者マッピングが反映される 🔵
  - **入力**: author / og:site_name を持つページの URL を共有
  - **期待結果**: カスタムフィールドに著者/発行者名が自動入力される

- [ ] **TC-042-06**: メタデータが存在しない場合は空文字列 🟡
  - **入力**: og:description が存在しないページの URL を共有
  - **期待結果**: カスタムフィールドの値が空文字列として表示される（エラーなし）

#### URL（URL 自動セット）

- [ ] **TC-043-01**: URL 値ソースフィールドに共有 URL が自動入力される 🔵
  - **前提**: URL 値ソースフィールドをデフォルトテンプレートに設定
  - **入力**: `https://example.com/article` を共有
  - **期待結果**: カスタムフィールドに `https://example.com/article` が自動入力される

- [ ] **TC-043-02**: テキスト共有時は URL フィールドが空文字列 🟡
  - **入力**: テキストを共有（URL なし）
  - **期待結果**: URL フィールドは空文字列

#### EMPTY（空値・ユーザー入力）

- [ ] **TC-044-01**: EMPTY フィールドが EditScreen に空で表示される 🔵
  - **前提**: EMPTY フィールド（キー名="mood"）をデフォルトテンプレートに設定
  - **入力**: URL を共有
  - **期待結果**: EditScreen に `mood:` のフィールドが空で表示され、入力可能
