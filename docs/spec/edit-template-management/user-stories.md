## エピック1: テンプレートの作成・管理

### ストーリー 1.1: テンプレートの一覧確認 🔵

**信頼性**: 🔵 *PRD「テンプレートを複数指定できる機能」・ユーザヒアリングより*

**私は** Share2Obsidian ユーザー **として**
**設定画面からテンプレート一覧を確認したい**
**そうすることで** 作成済みのテンプレートを把握できる

**関連要件**: REQ-001, REQ-011

**詳細シナリオ**:
1. SettingsScreen を開く
2. 「テンプレート管理」メニューをタップ
3. TemplateListScreen に遷移し、テンプレートの一覧が表示される
4. デフォルトテンプレートにはアイコン・バッジで識別表示される

**前提条件**:
- SettingsScreen が表示されていること（アイコンタップまたは共有フロー経由）

**制約事項**:
- テンプレートが 0 件の場合は「テンプレートなし」の空状態表示

**優先度**: Must Have

---

### ストーリー 1.2: テンプレートの新規作成 🔵

**信頼性**: 🔵 *PRD「テンプレートの編集画面」・ユーザヒアリングより*

**私は** Share2Obsidian ユーザー **として**
**新しいテンプレートを作成したい**
**そうすることで** 目的に合った front matter を持つノートを Obsidian に送れる

**関連要件**: REQ-012, REQ-015

**詳細シナリオ**:
1. TemplateListScreen の「+」ボタンをタップ
2. TemplateEditScreen に遷移（新規モード）
3. テンプレート名を入力
4. vault・folder を入力
5. カスタム front matter フィールドを追加（詳細はエピック2）
6. 保存ボタンをタップ
7. テンプレートが保存され TemplateListScreen に戻る

**前提条件**:
- TemplateListScreen が表示されていること

**優先度**: Must Have

---

### ストーリー 1.3: テンプレートのデフォルト設定 🔵

**信頼性**: 🔵 *ユーザヒアリング「デフォルト指定フラグ」「デフォルトテンプレートのみ自動適用」より*

**私は** Share2Obsidian ユーザー **として**
**よく使うテンプレートをデフォルトに設定したい**
**そうすることで** 共有するたびに自動的にそのテンプレートが適用される

**関連要件**: REQ-021, REQ-022, REQ-051

**詳細シナリオ**:
1. TemplateEditScreen を開く
2. 「デフォルトとして使用」トグルをオンにする
3. 保存する
4. 次回共有時、EditScreen にこのテンプレートのフィールドが自動反映される

**前提条件**:
- 少なくとも 1 件のテンプレートが存在すること

**制約事項**:
- デフォルトは同時に 1 つのみ（新たにデフォルト指定すると既存のデフォルトが自動解除）

**優先度**: Must Have

---

### ストーリー 1.4: テンプレートの削除 🟡

**信頼性**: 🟡 *CRUD 機能として設計文書から妥当な推測*

**私は** Share2Obsidian ユーザー **として**
**不要になったテンプレートを削除したい**
**そうすることで** テンプレート一覧を整理できる

**関連要件**: REQ-014, EDGE-002

**詳細シナリオ**:
1. TemplateListScreen でテンプレートの削除アイコンをタップ
2. 確認ダイアログが表示される
3. 確認後にテンプレートが削除される
4. デフォルトテンプレートを削除した場合は「デフォルトなし」状態になる（AppConfig フォールバック）

**優先度**: Should Have

---

## エピック2: Front matter フィールドの設定

### ストーリー 2.1: 固定値フィールドの追加 🔵

**信頼性**: 🔵 *ユーザヒアリング「固定値（ユーザーが手入力する初期値）」より*

**私は** Share2Obsidian ユーザー **として**
**固定の初期値を持つ front matter フィールドを追加したい**
**そうすることで** 毎回同じ値を手入力せずに済む

**関連要件**: REQ-031, REQ-033, REQ-041

**詳細シナリオ**:
1. TemplateEditScreen でフィールド追加ボタンをタップ
2. フィールドキー名を入力（例: `status`）
3. 値ソースを「固定値」に設定
4. 値の型を「文字列」または「リスト」に設定
5. 初期値を入力（例: `draft` または `web, reading`）
6. フィールドが追加され、編集画面に表示される

**優先度**: Must Have

---

### ストーリー 2.2: HTML メタデータマッピングフィールドの追加 🔵

**信頼性**: 🔵 *PRD「HTMLのメタデータを定義する機能」・ユーザヒアリングより*

**私は** Share2Obsidian ユーザー **として**
**URL/HTML 共有時にページの og:description をノートの `description` フィールドに自動入力したい**
**そうすることで** メタデータを手入力せずに構造化されたノートを作れる

**関連要件**: REQ-031, REQ-033, REQ-042, REQ-071

**詳細シナリオ**:
1. TemplateEditScreen でフィールド追加ボタンをタップ
2. フィールドキー名を入力（例: `description`）
3. 値ソースを「HTML メタデータ」に設定
4. マッピング元を「og:description / meta description」に設定
5. フィールドが追加される
6. このテンプレートで URL を共有すると、ページの og:description が自動的に `description` フィールドに入力される

**優先度**: Must Have

---

### ストーリー 2.3: ユーザー入力フィールドのプリセット追加 🔵

**信頼性**: 🔵 *ユーザヒアリング「空値（フィールド名だけ追加、値はユーザーが入力）」より*

**私は** Share2Obsidian ユーザー **として**
**フィールドキーだけを定義して実際の値は共有のたびに入力したい**
**そうすることで** 必要なフィールドが EditScreen に常に表示される**

**関連要件**: REQ-031, REQ-033, REQ-044

**詳細シナリオ**:
1. TemplateEditScreen でフィールド追加ボタンをタップ
2. フィールドキー名を入力（例: `mood`）
3. 値ソースを「空値（ユーザー入力）」に設定
4. フィールドが追加される
5. 共有時の EditScreen に `mood:` のフィールドが空で表示され、ユーザーが値を入力できる

**優先度**: Must Have

---

## エピック3: テンプレートの適用と EditScreen 連携

### ストーリー 3.1: デフォルトテンプレートによる自動適用 🔵

**信頼性**: 🔵 *ユーザヒアリング「デフォルトテンプレートのみ自動適用」より*

**私は** Share2Obsidian ユーザー **として**
**URL を共有したときにデフォルトテンプレートのフィールドが自動的に適用された状態で EditScreen を開きたい**
**そうすることで** 設定した front matter フィールドを毎回手動で追加せずに済む

**関連要件**: REQ-051, REQ-052, REQ-053

**詳細シナリオ**:
1. テンプレートに `source（URL）`, `description（og:description）`, `status（fixed: draft）` フィールドを設定しデフォルトにする
2. ブラウザで URL を共有
3. HTML 取得・処理完了後、EditScreen が開く
4. EditScreen のカスタムフィールドエリアに:
   - `source`: 共有 URL が自動入力
   - `description`: ページの og:description が自動入力
   - `status`: `draft` が初期値として表示
5. ユーザーは内容を確認・編集して送信

**優先度**: Must Have

---

### ストーリー 3.2: デフォルトテンプレート未設定時の動作（後方互換） 🔵

**信頼性**: 🔵 *REQ-023・既存実装・EDGE-001 より*

**私は** Share2Obsidian ユーザー **として**
**テンプレートを設定していなくても従来通りノートを共有したい**
**そうすることで** テンプレート機能を使わないユーザーも影響を受けない

**関連要件**: REQ-023, EDGE-001

**詳細シナリオ**:
1. テンプレートが 0 件またはデフォルトが未設定
2. URL/テキストを共有
3. EditScreen が開き、AppConfig の vault/folder/tags がデフォルト値として表示される
4. カスタムフィールドエリアは表示されない（空欄）

**優先度**: Must Have
