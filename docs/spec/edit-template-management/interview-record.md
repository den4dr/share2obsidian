### Q2: front matter フィールドの「値」として何を定義できる必要がありますか？

**質問日時**: 2026-05-31
**カテゴリ**: 未定義部分詳細化
**背景**: PRD「HTMLのメタデータや初期値を定義する機能」の具体的な値ソース種別が不明だった。

## ヒアリング結果サマリー

### 確認できた事項

- テンプレート管理は SettingsScreen から遷移する独立画面で実装する
- 値ソースは 4 種類（FIXED / HTML_META / URL / EMPTY）
- 共有時のテンプレート選択は「デフォルト自動適用」のみ（手動選択 UI 不要）
- HTML メタデータ対象: og:title, og:description, URL, 公開日, 最終更新日, 著者/発行者
- データ永続化: Room DB（初導入）
- FrontmatterBuilder の変更は許容
- テンプレートデータモデル: 名前, vault, folder, フィールド一覧, デフォルトフラグ
- 値型: STRING / LIST

### 追加/変更要件

- REQ-001: SettingsScreen → TemplateListScreen ナビゲーション（新規）
- REQ-041〜REQ-044: 4種類の値ソース仕様（新規）
- REQ-042: 6種類の HTML メタデータマッピング対象（新規）
- REQ-051: デフォルトテンプレート自動適用（新規）
- REQ-061〜REQ-063: Room DB 永続化（新規）
- REQ-071〜REQ-072: ProcessedContent へのメタデータ追加（新規）
- REQ-404: FrontmatterBuilder 拡張（新規）

### 残課題

- EDGE-005: カスタムフィールドキーが `title`/`tags` と重複した場合の動作が未定義
- テンプレートフィールドの表示順序変更（並べ替え）UI の必要性が未確認
- TemplateEditScreen でのバリデーション仕様（テンプレート名の最大文字数など）が未定義
