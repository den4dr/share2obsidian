## エピック1: アイコンからの起動

### ストーリー 1.1: アイコンタップで設定画面を開く 🔵

**信頼性**: 🔵 *PRD「共有以外の方法で普通にアイコンタップで起動できるようにする」・ユーザヒアリングより*

**私は** Share2Obsidian のユーザー **として**
**アプリアイコンをタップして設定画面を開きたい**
**そうすることで** 共有操作をしなくてもアプリを起動して設定を確認できる

**関連要件**: REQ-001, REQ-401

**詳細シナリオ**:
1. ホーム画面またはアプリドロワーで Share2Obsidian のアイコンをタップする
2. SettingsScreen が表示される（共有コンテンツなし）
3. 将来機能のプレースホルダーが表示される

**前提条件**:
- 共有インテントを持たない通常起動であること

**制約事項**:
- MainActivity 内で Compose 画面として表示（別 Activity 不使用）

**優先度**: Must Have

---

### ストーリー 1.2: 設定画面からアプリを終了する 🔵

**信頼性**: 🔵 *Android バックスタック仕様・ユーザヒアリング「戻るボタン」より*

**私は** アイコンから設定画面を開いたユーザー **として**
**バックボタンまたはツールバーの戻るボタンでアプリを終了したい**
**そうすることで** 設定画面を確認後にアプリを閉じられる

**関連要件**: REQ-102, REQ-103, EDGE-001

**詳細シナリオ**:
1. アイコンタップで SettingsScreen が表示されている
2. ツールバーの戻るボタン、またはバックボタンを押す
3. バックスタックが空のためアプリが終了する

**優先度**: Must Have

---

## エピック2: 共有フローからの遷移

### ストーリー 2.1: 共有中に設定画面へ移動する 🔵

**信頼性**: 🔵 *PRD「該当の画面へ共有中の画面内から飛べるようにする」・ユーザヒアリングより*

**私は** 別アプリからコンテンツを共有中のユーザー **として**
**EditScreen のツールバーにある設定アイコンをタップして設定画面へ移動したい**
**そうすることで** 共有操作の途中でも設定を確認できる

**関連要件**: REQ-002, REQ-101

**詳細シナリオ**:
1. 別アプリからコンテンツを Share2Obsidian へ共有する
2. EditScreen が表示される
3. トップバー右側の設定アイコンをタップする
4. SettingsScreen へ遷移する

**前提条件**:
- EditScreen が表示されている状態

**優先度**: Must Have

---

### ストーリー 2.2: 設定画面から EditScreen に戻る 🔵

**信頼性**: 🔵 *ユーザヒアリング「ツールバーの戻るボタンも追加」・EDGE-002 より*

**私は** EditScreen から設定画面へ移動したユーザー **として**
**戻るボタンで EditScreen に戻りたい**
**そうすることで** 設定を確認した後に共有操作を再開できる

**関連要件**: REQ-102, REQ-103, EDGE-002

**詳細シナリオ**:
1. EditScreen → SettingsScreen へ遷移している
2. ツールバーの戻るボタン、またはバックボタンを押す
3. EditScreen に戻り、編集中の内容が保持されている

**制約事項**:
- EditScreen の ViewModel 状態（フォーム入力内容）は戻った後も保持される

**優先度**: Must Have
