# 共有内容展開システム ヒアリング記録

**作成日**: 2026-03-28
**ヒアリング実施**: step4 既存情報ベースの差分ヒアリング

## ヒアリング目的

既存のShare2Obsidian実装（テキスト共有のみ対応）を分析し、「共有内容展開システム」として拡張すべき機能範囲・仕様を明確化するためのヒアリングを実施した。

---

## ヒアリング結果サマリー

### 確認できた事項

- コンテンツタイプ: テキスト + URL展開 + HTML変換 + 画像/ファイル（クリップボード）
- Obsidian URIパラメータ: content + title + vault + folder
- Frontmatter: title と tags（固定値）のみ
- Vault/フォルダ指定: コード内ハードコード（設定画面なし）
- URL展開手段: WebView
- HTML変換手段: ライブラリ（markwon等）
- エラーハンドリング: 現状から変更なし

### 追加/変更要件

- URL展開機能（REQ-101～103）: 新規追加
- HTML変換機能（REQ-201～203）: 新規追加
- ファイル/画像共有（REQ-301～303）: 新規追加
- Frontmatter追加（REQ-003）: 新規追加
- Obsidian URI拡張（REQ-002, 004, 005）: 変更

### 残課題

- ハードコードするVault名・Folder名の具体的な値は実装時に確認が必要
- WebViewでの本文抽出アルゴリズムの詳細（どのHTML要素を「本文」とみなすか）は実装時判断
- URL長上限超過時の対応（EDGE-101）はユーザー確認未実施（🔴）
