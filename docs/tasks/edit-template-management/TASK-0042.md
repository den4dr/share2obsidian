## 実装詳細

### 1. 統合テストシナリオの確認

**信頼性**: 🟡 *各フローから推測*

以下のフローを androidTest で確認する:

- **フロー1**: URL 共有 → Webメタデータ取得 → テンプレート適用 → EditScreen（カスタムフィールドあり）
- **フロー2**: テンプレート管理（新規作成 → デフォルト設定）
- **フロー3**: カスタムフィールドあり → 送信 → Frontmatter 確認
- **フロー5**: テンプレートなし → 従来動作（カスタムフィールドセクション非表示）

**テストファイル**: `app/src/androidTest/java/com/den4dr/share2Obsidian/integration/`

---

### 2. Lint チェック

**信頼性**: 🟡 *品質確認から推測*

`mise exec -- ./gradlew lint` でエラーなし確認する。未使用インポート・変数を削除する。

---

### 3. エッジケース確認

**信頼性**: 🔵 *要件定義 EDGE-001〜005より*

以下のエッジケースが正しく動作することを確認する:

- **EDGE-001**: デフォルトテンプレートなし → AppConfig フォールバック
- **EDGE-002**: テンプレート削除でフィールドも削除（DB CASCADE）
- **EDGE-003**: WebView タイムアウト → metadata = emptyMap()
- **EDGE-004**: HTML 共有時 → sourceUrl = null
- **EDGE-005**: カスタムフィールドの key="tags" → 標準 tags 上書き
