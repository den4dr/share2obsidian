## 実装詳細

### 1. MainActivity へのテンプレート適用ロジック追加

**信頼性**: 🔵 *architecture.md テンプレート適用フロー・dataflow.md フロー1より*

`@AndroidEntryPoint` アノテーションは TASK-0028 で追加済みのため、`@Inject` による `TemplateRepository` の注入のみ追加する。

共有フロー処理内（`lifecycleScope.launch`）:

```kotlin
// コンテンツ処理後
val defaultTemplate = templateRepository.getDefaultTemplate()

val config = if (defaultTemplate != null) {
    NoteConfig(
        vault = defaultTemplate.vault,
        folder = defaultTemplate.folder,
        defaultTags = AppConfig.OBSIDIAN_TAGS
    )
} else {
    NoteConfig.fromAppConfig()  // EDGE-001: AppConfig フォールバック
}

val customFields = defaultTemplate?.fields?.map { field ->
    val value = when (field.valueSource) {
        FieldValueSource.FIXED     -> field.defaultValue
        FieldValueSource.HTML_META -> processed.metadata[field.metaKey] ?: ""
        FieldValueSource.URL       -> processed.sourceUrl ?: ""
        FieldValueSource.EMPTY     -> ""
    }
    CustomFieldState(field.key, value, field.valueType)
} ?: emptyList()

viewModel.initialize(processed, config, customFields)
```

**実装ファイル**: `app/src/main/java/com/den4dr/share2Obsidian/MainActivity.kt`（既存ファイルを変更）
