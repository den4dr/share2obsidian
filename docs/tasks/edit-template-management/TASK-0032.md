## タスク概要

`TemplateEditScreen` Composable の基本情報 UI を実装する。テンプレート名・vault・folder・デフォルトフラグの入力フォーム、「保存」ボタン、保存後の自動ナビゲーション（LaunchedEffect + isSaved）、および「戻る」ボタンを含む。Compose UI テストも作成する。

## 実装詳細

### 1. TemplateEditScreen の基本構造

**信頼性**: 🔵 *architecture.md REQ-012〜015より*

```kotlin
@Composable
fun TemplateEditScreen(
    templateId: Long?,  // null=新規, Long=既存テンプレートID
    viewModel: TemplateEditViewModel = hiltViewModel(),
    onNavigateBack: () -> Unit,
)
```

- TopAppBar: 「テンプレート作成」または「テンプレート編集」+ 戻るボタン
- テンプレート名入力 (OutlinedTextField)
- vault 入力 (OutlinedTextField)
- folder 入力 (OutlinedTextField)
- デフォルト設定 (Switch + ラベル)
- 「保存」ボタン (Button, Modifier.fillMaxWidth())
- LaunchedEffect で uiState.isSaved が true になったら onNavigateBack() を呼ぶ

**実装ファイル**: `app/src/main/java/com/den4dr/share2Obsidian/ui/template/TemplateEditScreen.kt`
