### 2. MainActivity の共有フローへの状態変数追加

**信頼性**: 🔵 *architecture.md ナビゲーション設計より*

```kotlin
var showTemplateList by rememberSaveable { mutableStateOf(false) }
var editingTemplateId by rememberSaveable { mutableStateOf<Long?>(null) }

when {
    editingTemplateId != null ->
        TemplateEditScreen(
            templateId = editingTemplateId,
            onNavigateBack = { editingTemplateId = null }
        )
    showTemplateList ->
        TemplateListScreen(
            onNavigateBack = { showTemplateList = false },
            onNavigateToEdit = { id -> editingTemplateId = id }
        )
    showSettings ->
        SettingsScreen(
            onNavigateBack = { showSettings = false },
            onNavigateToTemplates = { showTemplateList = true }
        )
    else ->
        EditScreen(...)
}
```

**実装ファイル**: `app/src/main/java/com/den4dr/share2Obsidian/MainActivity.kt`（既存ファイルを変更）

## 注意事項

- editingTemplateId は null=テンプレート画面なし、Long=TemplateEditScreen を表示
- TemplateListScreen からの新規作成: onNavigateToEdit(null) → editingTemplateId = null（TemplateEditScreen は templateId=null で新規モード）
- TemplateListScreen からの既存編集: onNavigateToEdit(id) → editingTemplateId = id
- Jetpack Navigation は不使用。`rememberSaveable { mutableStateOf() }` によるネスト状態管理（設計ヒアリング確認済み）
- Gradleコマンドは必ず `mise exec -- ./gradlew ...` で実行すること
