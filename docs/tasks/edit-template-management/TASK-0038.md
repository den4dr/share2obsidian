## タスク概要

`NoteComposer.buildFrontmatter` に `customFields: List<CustomFieldState>` パラメータを追加し、カスタムフィールドを標準 tags より先に Frontmatter へ出力する。カスタムフィールドに key="tags" が存在する場合は標準 tags 行を出力しない（EDGE-005: カスタムが上書き）。

## 実装詳細

### 1. NoteComposer.buildFrontmatter 拡張

**信頼性**: 🔵 *architecture.md NoteComposer拡張設計・EDGE-005より*

```kotlin
fun buildFrontmatter(
    body: String,
    tags: List<String>,
    customFields: List<CustomFieldState> = emptyList()
): String {
    val sb = StringBuilder("---\n")
    val customKeySet = customFields.map { it.key }.toSet()

    // カスタムフィールドを先に出力
    for (field in customFields) {
        val valueStr = if (field.valueType == FieldValueType.LIST) {
            "[${field.value}]"
        } else {
            field.value
        }
        sb.append("${field.key}: $valueStr\n")
    }

    // "tags" がカスタムフィールドに含まれない場合のみ標準 tags を出力
    if ("tags" !in customKeySet) {
        val tagsString = tags.joinToString(", ")
        sb.append("tags: [$tagsString]\n")
    }

    sb.append("---\n\n$body")
    return sb.toString()
}
```
