## タスク概要

EditScreen の表示順を「vault → folder → title（ファイル名）→ frontmatter フィールド → 本文」に変更し、vault/folder の入力欄を追加する。title はファイル名として frontmatter とは視覚的に別セクションに表示する。vault/folder の変更は `viewModel.updateVault/updateFolder` に接続する。

## 依存タスク

- **前提タスク**: [TASK-0048](TASK-0048.md)（updateVault・EditFormState.vault）
- **後続タスク**: [TASK-0054](TASK-0054.md)

## 完了条件

- [ ] 表示順が vault → folder → title → frontmatter → body になっている
- [ ] vault 入力欄が最上部に追加されている（REQ-043）
- [ ] folder 入力欄が vault の次に表示される
- [ ] title 欄がファイル名として frontmatter セクションと視覚的に分離されている（REQ-042）

**信頼性**: 🔵 *REQ-041・dataflow.md フロー1 表示順より*

  2. folder 入力欄（位置変更）
  3. title 入力欄（ファイル名として明示、別セクション）
  4. frontmatter フィールド（customFields）
  5. body 入力欄（最下部）

### 2. vault/folder 入力欄の追加 🔵

**信頼性**: 🔵 *REQ-043・TC-041-01, TC-041-02 より*

```kotlin
OutlinedTextField(
    value = formState.vault,
    onValueChange = viewModel::updateVault,
    label = { Text(stringResource(R.string.edit_label_vault)) },
    singleLine = true,
    modifier = Modifier.fillMaxWidth().testTag("vault_field"),

// folder も同様（updateFolder へ接続）
```

### 3. title と frontmatter の視覚的分離 🔵

**信頼性**: 🔵 *REQ-042・TC-041-04 より*
