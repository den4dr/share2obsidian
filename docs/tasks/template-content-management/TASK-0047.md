## タスク概要

`TemplateApplicator` を Template 非依存の設定構築と本文テンプレート解決に対応させる。`buildConfig(settings: NoteSettings)` で DataStore 由来の vault/folder から `NoteConfig` を構築し、`buildBody(template, sharedBody)` で `{{content}}` プレースホルダーを解決する。`buildCustomFields` は変更しない。

## 実装詳細

### 1. buildConfig の変更 🔵

**信頼性**: 🔵 *architecture.md TemplateApplicator・REQ-031 より*

**実装ファイル**: `app/src/main/java/com/den4dr/share2Obsidian/TemplateApplicator.kt`

```kotlin
fun buildConfig(settings: NoteSettings): NoteConfig = NoteConfig(
    vault = settings.vault,
    folder = settings.folder,
    defaultTags = AppConfig.OBSIDIAN_TAGS,
)
```

> 旧 `buildConfig(template)` は削除。Template から vault/folder を取得しない（REQ-031）。

### 2. buildBody の新規追加 🔵

**信頼性**: 🔵 *architecture.md/interfaces.kt buildBody・dataflow.md フロー5 より*

```kotlin
fun buildBody(template: Template?, sharedBody: String): String {
    val templateBody = template?.body ?: ""
    return if (templateBody.isEmpty()) sharedBody              // REQ-013, REQ-014
           else templateBody.replace("{{content}}", sharedBody) // REQ-012, EDGE-001/002
}
```

### 3. buildCustomFields 維持 🔵

**信頼性**: 🔵 *architecture.md「変更なし」より*

`buildCustomFields(template, processed)` のシグネチャ・ロジックは変更しない。
