## 実装詳細

### 1. E2E シナリオの検証 🔵

**信頼性**: 🔵 *user-stories.md 全ストーリー・dataflow.md フロー1〜7 より*

**確認シナリオ**:
1. URL 共有 → デフォルトテンプレート（body に `{{content}}`）適用 → EditScreen 本文に解決済み構造が表示
2. EditScreen 表示順（vault→folder→title→frontmatter→body）の確認
3. vault/folder を変更して送信 → URI に反映、DataStore は不変
4. SettingsScreen で vault/folder を設定 → EditScreen 初期値に反映
5. TemplateEditScreen で body テンプレート作成・保存（vault/folder 欄なし）
6. Room v1→v2 マイグレーションで既存データ保持

### 2. Edge ケースの確認 🔵

**信頼性**: 🔵 *acceptance-criteria.md Edge ケース TC-011-E01/E02/E03, TC-021-E01 より*

- `{{content}}` 複数置換、`{{content}}` なし非空 body、テキスト共有時の置換、vault/folder 未設定時の空欄表示

### 3. リグレッション確認 🟡

**信頼性**: 🟡 *既存機能（edit-template-management）への影響確認は妥当な推測を含む*

- カスタムフィールド・frontmatter 出力・HTML メタデータ抽出など既存機能が壊れていないこと
