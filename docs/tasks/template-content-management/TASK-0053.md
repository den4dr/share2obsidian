## タスク概要

MainActivity を新しい適用フローへ統合する。`NoteSettingsRepository` を inject し、`buildConfig(settings)` で DataStore 由来の config を構築、`buildBody(template, processed.body)` で本文を解決して `viewModel.initialize` に渡す。`buildSendParams()` の引数削除に伴う `onSend` 呼び出しを修正する。

- **前提タスク**: [TASK-0045](TASK-0045.md)（Repository）, [TASK-0046](TASK-0046.md)（NoteSettingsRepository）, [TASK-0047](TASK-0047.md)（buildConfig/buildBody）, [TASK-0048](TASK-0048.md)（buildSendParams 引数削除）
- **後続タスク**: [TASK-0054](TASK-0054.md)

## 完了条件

- [ ] `NoteSettingsRepository` が MainActivity に inject されている
- [ ] `buildConfig(noteSettings)` で config を構築している（Template から vault/folder を取得しない）

- [ ] `buildSendParams()`（引数なし）で onSend を呼んでいる
- [ ] 設定取得・テンプレート取得が並行/適切な順序で実行される

### 2. 適用フローの統合 🔵

**信頼性**: 🔵 *dataflow.md フロー1 ②〜⑥ より*

```kotlin
val noteSettings = noteSettingsRepository.getSettings().first()       // ②
val template = templateRepository.getDefaultTemplate().first()        // ②
val config = TemplateApplicator.buildConfig(noteSettings)            // ③ REQ-031
val resolvedBody = TemplateApplicator.buildBody(template, processed.body) // ④ REQ-032
val customFields = TemplateApplicator.buildCustomFields(template, processed) // ⑤
viewModel.initialize(processed.copy(body = resolvedBody), config, customFields) // ⑥
```

### 3. onSend の buildSendParams 修正 🔵

**信頼性**: 🔵 *dataflow.md フロー2・REQ-062 より*

// 旧: viewModel.buildSendParams(config)

NoteComposer.buildFrontmatter(sendParams.body, sendParams.tags, sendParams.customFields)
