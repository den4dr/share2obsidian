## タスク概要

EditScreen が vault/folder を保存ごとに編集できるよう、状態層を拡張する。`EditFormState` に `vault: String` を追加し、`EditScreenViewModel` の `initialize` で config.vault から初期化、`updateVault()` を追加、`buildSendParams()` から config 引数を削除して EditFormState の vault/folder から `NoteConfig` を構築する。`SendParams` は意味を明確化する（vault/folder は EditFormState 由来）。

### 2. EditScreenViewModel の変更 🔵

**信頼性**: 🔵 *interfaces.kt EditScreenViewModel・dataflow.md フロー2 より*

**実装ファイル**: `app/src/main/java/com/den4dr/share2Obsidian/ui/EditScreenViewModel.kt`

```kotlin
fun initialize(processed: ProcessedContent, config: NoteConfig, customFields: List<CustomFieldState>) {
    if (initialized) return
    initialized = true
    _formState.value = EditFormState(
        vault = config.vault,       // 新規
        folder = config.folder,
        title = processed.title ?: "",
        body = processed.body,
        tagsText = config.defaultTags.joinToString(", "),
        customFields = customFields,
    )
}

fun updateVault(vault: String) {                 // 新規追加
    _formState.value = _formState.value.copy(vault = vault)
}

fun buildSendParams(): SendParams {              // 引数削除
    val state = _formState.value
    return SendParams(
        title = state.title.ifBlank { null },
        body = state.body,
        tags = parseTagsText(state.tagsText),
        config = NoteConfig(vault = state.vault, folder = state.folder, defaultTags = emptyList()),
        customFields = state.customFields,
    )
}
```
