## タスク概要

`MainActivity` のナビゲーションロジックを変更する。

1. **直接起動（Intent なし）**: `setContent { SettingsScreen(onNavigateBack = { finish() }) }` を即時呼び出し
2. **共有フロー**: `setContent` 内に `rememberSaveable { mutableStateOf(false) }` による `showSettings` 状態を追加し、EditScreen ↔ SettingsScreen の条件分岐を実装

## 実装詳細

### 1. 直接起動フローの実装 🔵

**信頼性**: 🔵 *REQ-001・architecture.md「起動経路1」より*

**実装ファイル**: `app/src/main/java/com/den4dr/share2Obsidian/MainActivity.kt`

```kotlin
val shareContent = ContentTypeDetector.detect(intent)
if (shareContent == null) {
    // 直接起動 → SettingsScreen を即表示
    setContent {
        Theme.Share2ObsidianTheme {
            SettingsScreen(onNavigateBack = { finish() })
        }
    }
    return
}
```

### 2. 共有フローへの showSettings 状態追加 🔵

**信頼性**: 🔵 *REQ-101・architecture.md「起動経路2」・EDGE-101（rememberSaveable）より*

```kotlin
setContent {
    Theme.Share2ObsidianTheme {
        var showSettings by rememberSaveable { mutableStateOf(false) }

        if (showSettings) {
            SettingsScreen(onNavigateBack = { showSettings = false })
        } else {
            EditScreen(
                viewModel = viewModel,
                config = config,
                onSend = { sendParams -> /* 既存処理（変更なし）*/ },
                onCancel = { finish() },
                onNavigateToSettings = { showSettings = true },
            )
        }
    }
}
```

**重要**: `rememberSaveable` を使用することで画面回転（EDGE-101）後も `showSettings` 状態が復元される。

## 注意事項

- `rememberSaveable` は `@Composable` 関数内でのみ使用可能。`setContent { }` のラムダ内で定義すること
- 直接起動フローの `setContent` は `lifecycleScope.launch` の外（同期処理）で呼ぶこと
- 共有フローの `setContent` は `lifecycleScope.launch` 内（非同期処理完了後）で呼ぶこと
- `onSend` の内部処理（NoteComposer 呼び出し → startActivity → finish()）は変更しない
